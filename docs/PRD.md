## 一、产品概述

### 1.1 产品定位

**KidsFit 小勇士** 是一款面向3-12岁儿童及其家长的AI运动健康App，以"科学运动 + 趣味互动 + 亲子参与"为核心价值主张。产品通过AI骨骼识别、游戏化训练、视力健康管理、技能进阶体系和正向竞技心理培养，帮助儿童在快乐中建立运动习惯、提升身体素质、保护视力健康，并塑造积极的体育精神。

### 1.2 目标用户

| 用户角色 | 年龄段 | 核心诉求 | 使用场景 |
|---------|--------|---------|---------|
| **儿童用户（主用户）** | 3-12岁 | 好玩、有成就感、能和家长一起玩 | 每日运动打卡、完成训练任务、挑战游戏关卡 |
| **家长用户（决策者/付费者）** | 25-45岁 | 了解孩子运动数据、管理视力健康、促进亲子关系 | 查看数据看板、定制训练计划、参与亲子挑战 |

### 1.3 产品价值主张

- **对儿童**：让运动像玩游戏一样有趣，每次运动都有进步和奖励
- **对家长**：科学了解孩子体质发展，省心管理视力健康，高质量亲子互动
- **差异化**：AI实时动作指导 + 视力健康闭环管理 + 成长型竞技心理培养

### 1.4 成功指标（OKR）

| 目标 | 关键结果 | 衡量标准 |
|-----|---------|---------|
| 用户活跃度 | 日活跃儿童用户占比 | DAU/MAU ≥ 35% |
| 运动习惯养成 | 周运动打卡完成率 | ≥ 60% |
| 视力健康改善 | 日均户外运动时长 | ≥ 90分钟 |
| 家长满意度 | NPS评分 | ≥ 50 |
| 留存率 | 次月留存率 | ≥ 45% |

## 三、功能需求详解

### 3.1 运动协调性与灵活性模块

#### 3.1.1 AI骨骼实时评分系统

**功能描述**：通过摄像头实时捕捉儿童运动姿态，进行骨骼关键点识别与动作质量评估。

**输入**：
- 摄像头实时视频流（30fps）
- 用户年龄（用于体型参数校准）

**处理流程**：
1. 视频帧预处理（降噪、对比度增强）
2. ML Kit Pose Detection 提取33个骨骼关键点
3. 关键角度计算（关节角度、肢体对称性）
4. 动作轨迹分析（速度、加速度、连贯性）
5. 与标准动作模板对比评分

**输出**：
- 综合评分（0-100分）
- 四个维度评分：节奏准确性、幅度规范性、身体对称性、连贯性
- 实时纠正建议（语音 + 文字提示）
- 动作回放与对比分析

**年龄适配参数**：

| 参数项 | 3-6岁 | 7-9岁 | 10-12岁 |
|-------|-------|-------|---------|
| 骨骼比例基准 | 头大身短 | 逐渐均衡 | 接近成人比例 |
| 动作幅度容忍度 | ±30% | ±20% | ±15% |
| 节奏容错（BPM偏差） | ±15 | ±10 | ±8 |
| 对称性权重 | 0.2 | 0.25 | 0.3 |
| 最低有效检测置信度 | 0.5 | 0.6 | 0.7 |

#### 3.1.2 节奏协调训练

**音乐跳跃**：
- 跟随音乐节奏进行跳跃动作
- 节奏匹配度实时评分
- 难度分级：慢速（60BPM）→ 中速（100BPM）→ 快速（140BPM）

**跑酷游戏**：
- 横向移动躲避障碍物
- 跳跃/下蹲通过不同高度障碍
- 收集金币增加积分
- 关卡制，每关增加速度和障碍复杂度

#### 3.1.3 平衡挑战

**功能描述**：利用陀螺仪和摄像头结合，进行平衡能力训练。

**模式一：单脚站立挑战**
- 摄像头检测单脚站立姿态
- 陀螺仪检测身体晃动幅度
- 目标：保持单脚站立30/60/90秒
- 晃动越小，得分越高

**模式二：虚拟走钢丝**
- 手机横握，通过倾斜控制虚拟角色平衡
- 摄像头检测身体姿态辅助评分
- 路径越长、越窄，难度越高

#### 3.1.4 敏捷反应训练

**功能描述**：通过视觉提示训练反应速度和变向能力。

**模式一：颜色指令跑**
- 屏幕显示颜色指令（红=停、绿=跑、黄=跳）
- 摄像头检测对应动作
- 反应时间越短得分越高

**模式二：方向箭头**
- 屏幕显示方向箭头
- 儿童需快速向对应方向移动
- 连续正确获得连击加分

#### 3.2.2 每日智能推荐引擎

**输入参数**：
```json
{
  "age": 8,
  "fitness_assessment": {
    "endurance": 6,
    "agility": 7,
    "strength": 5,
    "speed": 6,
    "coordination": 8,
    "balance": 7,
    "flexibility": 6
  },
  "history": {
    "weekly_duration_minutes": 120,
    "weekly_frequency": 5,
    "completion_rate": 0.75,
    "last_trained_muscle_groups": ["legs", "core"]
  },
  "preferences": ["jump_rope", "dance"],
  "weather": {
    "is_outdoor_suitable": true,
    "temperature": 22,
    "aqi": 45
  },
  "eye_health": {
    "daily_outdoor_minutes": 60,
    "target_outdoor_minutes": 120
  }
}
```

**推荐逻辑**：

```
算法：每日训练计划生成

输入：用户画像 P = (age, fitness, history, prefs, weather, eye)
输出：训练计划 Plan = (warmup[], main[], cooldown[], total_duration)

1. 确定训练总量
   base_duration = age_based_duration(age)  // 3-6:15, 7-9:20, 10-12:25
   adjust = 0
   if history.completion_rate > 0.8: adjust += 5
   if history.completion_rate < 0.5: adjust -= 5
   if weather.is_outdoor_suitable && eye.daily_outdoor < eye.target:
       adjust += 5  // 鼓励户外运动
   total_duration = clamp(base_duration + adjust, 15, 30)

2. 确定训练结构
   warmup_ratio = 0.2
   main_ratio = 0.6
   cooldown_ratio = 0.2
   
   warmup_duration = total_duration * warmup_ratio
   main_duration = total_duration * main_ratio
   cooldown_duration = total_duration * cooldown_ratio

3. 选择热身动作（低强度、全身激活）
   warmup_pool = filter_by_age(warmup_exercises, age)
   warmup = select_diverse(warmup_pool, warmup_duration, target_muscle="full_body")

4. 选择主训练动作
   // 优先训练弱项
   weak_dimensions = sort_by_score(fitness)[:2]
   // 避免连续训练相同肌群
   available_exercises = filter_out_recent_muscle_groups(exercises, history.last_trained)
   // 结合偏好
   preferred_exercises = filter_by_preferences(available_exercises, prefs)
   // 天气适合户外时优先户外运动
   if weather.is_outdoor_suitable:
       preferred_exercises = prioritize_outdoor(preferred_exercises)
   
   main = build_circuit(preferred_exercises, main_duration, weak_dimensions)

5. 选择拉伸动作
   cooldown = select_stretching(main, cooldown_duration)

6. 组装计划
   return Plan(warmup, main, cooldown, total_duration)
```

#### 3.2.3 AI运动打卡系统

**支持运动类型**：
- 跳绳（计数 + 节奏分析）
- 开合跳（计数 + 幅度检测）
- 深蹲（计数 + 膝盖角度检测）
- 仰卧起坐（计数 + 起身幅度检测）
- 原地高抬腿（计数 + 抬腿高度检测）
- 俯卧撑（计数 + 身体平直度检测）

**计数算法**：
```
跳绳计数：
- 检测手腕关键点垂直位移
- 一个完整周期 = 最低点 → 最高点 → 最低点
- 结合脚踝位移验证（双脚离地）
- 排除假阳性：周期时长需在合理范围（0.3-2秒）

深蹲计数：
- 检测髋关节和膝关节角度
- 下蹲：髋角 < 100° 且 膝角 < 90°
- 站起：髋角 > 160° 且 膝角 > 160°
- 一个完整周期 = 站立 → 下蹲 → 站立

开合跳计数：
- 检测双手和双脚的水平间距
- 跳开：手脚间距 > 肩宽1.5倍
- 跳合：手脚间距 < 肩宽0.5倍
- 一个完整周期 = 合 → 开 → 合
```

**实时纠正提示库**：

| 运动 | 常见问题 | 检测条件 | 语音提示 |
|-----|---------|---------|---------|
| 跳绳 | 手臂张开过大 | 肘角 > 120° | "手臂再收拢一点，用手腕摇绳" |
| 跳绳 | 跳得太高 | 重心垂直位移 > 15cm | "轻轻跳，脚尖着地就好" |
| 深蹲 | 膝盖内扣 | 膝盖水平位置 < 脚尖水平位置 | "膝盖打开，对准脚尖方向" |
| 深蹲 | 蹲得不够低 | 髋角 > 110° | "再往下蹲一点，像坐椅子一样" |
| 开合跳 | 节奏不稳 | 周期时长变异系数 > 0.3 | "保持节奏，一二一二" |
| 仰卧起坐 | 借力 | 手肘与膝盖距离变化 < 5cm | "用肚子用力，不要用手拉" |

#### 3.2.4 体测模拟与达标分析

**体测项目（按年龄）**：

| 年龄 | 测试项目 | 达标标准示例 |
|-----|---------|------------|
| 3-6岁 | 单脚站立、立定跳远、走平衡木 | 单脚站5秒、跳远50cm |
| 7-9岁 | 50米跑、坐位体前屈、1分钟跳绳 | 50米<11秒、跳绳60个 |

**达标分析**：

- 生成能力雷达图（耐力、力量、速度、灵敏、柔韧、协调）
- 提供针对性提升建议

#### 3.2.5 成长数据看板

**数据项**：

- 运动能力：各项体测成绩趋势

**可视化**：

- 能力雷达图历史对比

### 3.3 用眼卫生与好视力模块

#### 3.3.1 户外运动时长自动追踪

**识别逻辑**：
```
户外运动判定条件（需同时满足）：
1. GPS定位：速度 > 0.5m/s 且 位置变化 > 50米/小时
2. 环境光：光照强度 > 1000 lux（户外典型值）
3. 加速度计：活动量 > 静止阈值
4. 时间窗口：白天时段（6:00-18:00）

室内运动判定：
- GPS位置稳定（变化 < 20米/小时）
- 或光照强度 < 500 lux

状态切换：
- 进入户外：连续5分钟满足户外条件 → 开始计时
- 离开户外：连续10分钟不满足 → 停止计时
```

**每日目标**：
- 推荐户外时间：120分钟
- 最低有效户外时间：60分钟
- 达标提醒：达到目标时推送"今日户外运动量已达标！"

#### 3.3.2 用眼提醒系统

**20-20-20法则提醒**：
- 触发条件：连续近距离用眼（屏幕使用）20分钟
- 提醒内容："已经看了20分钟啦，快看看20英尺（6米）外的东西，休息20秒吧！"
- 配合计时器引导远眺

**室内超时提醒**：
- 触发条件：连续室内时间 > 45分钟
- 提醒内容："已经在室内很久啦，去窗边看看远方吧！"
- 提供"极目远眺"快速训练入口

**用眼疲劳监测**：
- 结合屏幕使用时长、环境光变化、眨眼频率（可选）
- 生成每日用眼负荷评分

#### 3.3.3 视力档案管理

**验光单OCR**：
- 支持上传验光单照片
- OCR识别字段：球镜（SPH）、柱镜（CYL）、轴位（AXIS）、矫正视力（VA）
- 眼轴长度（AL）手动录入或OCR识别
- 数据校验：异常值提示复核

**视力变化趋势**：
- 屈光度变化曲线（球镜、柱镜）
- 眼轴长度增长曲线
- 远视储备计算与预警

**远视储备预警**：
```
远视储备正常值（按年龄）：
- 3岁：+2.00D ~ +3.00D
- 6岁：+1.50D ~ +2.00D
- 9岁：+0.75D ~ +1.25D
- 12岁：+0.25D ~ +0.75D

预警逻辑：
- 绿色：储备充足（高于年龄下限）
- 黄色：储备不足（接近年龄下限）→ 推送强化户外提醒
- 红色：储备耗尽（低于年龄下限或已近视）→ 推送就医建议
```

**数据安全**：
- 视力数据加密存储
- 仅家长端可查看完整档案
- 儿童端仅显示"视力状态图标"（笑脸/中性/关注）

#### 3.3.4 护眼运动专项

**极目远眺训练**：
- 语音引导："找到窗外最远的地方，用力看清楚，保持5秒..."
- 计时器：20秒为一个周期
- 每日建议：3-5个周期

**晶体操**：
- 远近交替注视：手指近处（30cm）→ 远处（6米外）
- 配合语音节奏引导
- 动画演示正确方法

#### 3.3.5 户外运动场所推荐

**推荐逻辑**：
- 基于GPS定位推荐附近公园、操场、运动场馆
- 过滤条件：适合儿童、有运动设施、安全性高
- 显示距离、导航入口、其他用户评价
- 天气适宜时优先推荐户外场所

#### 3.4.2 AI动作拆解教学

1. **示范阶段**：标准动作视频播放，关键帧暂停讲解要点
2. **跟练阶段**：儿童跟随视频做动作，AI实时检测并评分
3. **对比阶段**：儿童动作回放与标准动作并排对比，标注差异点

**关键帧标注**：
- 在标准动作视频中标记关键姿势帧

#### 3.4.4 技能成就树

### 3.5 健康竞技对抗心理模块

#### 3.5.1 模拟竞技场景

**游戏化竞技体验**：
- **虚拟赛跑**：摄像头检测跑步动作，屏幕显示虚拟赛道和对手
- **投篮大赛**：手势模拟投篮，检测出手角度和力度

**AI对手难度**：
- 简单：固定成绩，儿童容易获胜
- 中等：根据儿童历史成绩动态调整
- 困难：略高于儿童最佳成绩，提供挑战

#### 3.5.2 亲子挑战赛

**智能让分匹配算法**：
```
输入：
- 家长能力值 P（基于历史成绩或体测数据）
- 儿童能力值 C
- 家长选择的让分强度 S（轻度/中度/重度）

让分计算：
差距 = P - C

轻度让分：
  儿童目标 = 家长目标 × 0.85
  
中度让分：
  儿童目标 = 家长目标 × 0.7
  
重度让分：
  儿童目标 = 家长目标 × 0.55

动态调整（3局后）：
  if 儿童连胜3局: 减少让分比例 5%
  if 家长连胜3局: 增加让分比例 5%
  
输出：双方各自的目标成绩，胜负判定标准
```

**挑战模式**：
- 同步挑战：双方同时运动，实时比分
- 异步挑战：一方先完成，另一方挑战记录
- 限时挑战：规定时间内完成尽可能多的次数

#### 3.5.3 失败正向反馈系统

**核心原则**：
- 强调"和自己比"，每次尝试都是进步
- 失败时绝不使用负面评价
- 关注过程中的亮点

**鼓励语音库（20+条）**：

| 场景 | 鼓励语 |
|-----|-------|
| 未达标但比上次好 | "比上次进步了！继续加油！" |

| 首次尝试 | "第一次能做到这样已经很厉害了！" |
| 接近目标 | "差一点就成功了，下次一定行！" |
| 节奏变好 | "这次节奏把握得很好！" |
| 幅度进步 | "比上次跳得更高了！" |
| 连续尝试 | "你这么努力练习，一定会越来越棒的！" |
| 亲子挑战输 | "爸爸妈妈也是练了很久的，你再多练习就能追上啦！" |
| 打破自己的记录 | "哇！创造了新纪录！你是自己的冠军！" |

**鼓励动画**：

| 指标 | 要求 | 说明 |

- **最低Android版本**：Android 8.0 (API 26)
- **目标Android版本**：Android 14 (API 34)
- **设备要求**：
  - 摄像头：后置≥800万像素，支持30fps

- **数据加密**：
  - 本地敏感数据（视力数据、体征数据）AES-256加密
  - 网络传输TLS 1.3
  - 云端存储加密
- **隐私保护**：
  - 摄像头数据实时处理，不存储原始视频
  - 运动照片/视频仅本地存储
  - 儿童个人信息最小化收集（不收集姓名、学校等）

- **合规要求**：
  - 《儿童个人信息网络保护规定》
  - 《个人信息保护法》
  - 家长知情同意机制
  - 数据删除权（家长可随时删除全部数据）

- **儿童端**：
  - 大按钮（≥80×80dp）

  - 数据可视化清晰

- ANR率 < 0.05%

## 六、UI/UX设计规范

### 6.1 设计原则

- **明亮活泼**：高饱和度色彩，营造积极向上的氛围
- **简洁直观**：大图+大按钮+少文字，降低认知负担
- **即时反馈**：每次操作都有视觉/听觉反馈
- **正向引导**：鼓励为主，避免负面提示
- **年龄适配**：界面复杂度随年龄递增

### 6.2 色彩方案

**主色调**：
- 活力橙：#FF6B35（主按钮、重点强调）
- 天空蓝：#4ECDC4（次要按钮、背景）
- 阳光黄：#FFE66D（高亮、警告、成就）
- 健康绿：#96CEB4（成功、完成、户外）
- 温暖红：#FF6B6B（错误、重要提醒）

**中性色**：
- 深灰：#2C3E50（文字）
- 中灰：#7F8C8D（次要文字）
- 浅灰：#ECF0F1（背景、分隔线）
- 白色：#FFFFFF（卡片背景）

**年龄适配色彩**：
- 3-6岁：更多暖色调（橙、黄），减少冷色
- 7-12岁：可使用完整色板，增加蓝色/绿色占比

### 6.3 字体规范

- **中文主字体**： rounded字体（如站酷快乐体、方正兰亭圆体）
- **最小字号**：
  - 3-6岁：28sp（无大量文字）
  - 7-9岁：24sp
  - 10-12岁：20sp
- **标题字号**：32-48sp
- **按钮文字**：24-28sp

### 6.4 组件规范

**按钮**：
- 主按钮：圆角16dp，高度64dp，阴影4dp
- 次按钮：圆角12dp，高度56dp，描边2dp
- 图标按钮：80×80dp，圆角20dp

**卡片**：
- 圆角20dp
- 阴影2dp
- 内边距16dp

**输入框**：
- 圆角12dp
- 高度56dp
- 聚焦状态：主色边框

### 6.5 动画规范

- **页面切换**：滑动过渡，300ms，ease-out
- **按钮点击**：缩放0.95，100ms
- **成就弹出**：从底部滑入 + 弹跳效果，500ms
- **加载状态**：角色IP跑步/跳跃循环动画
- **成功反馈**：彩带粒子效果 + 角色庆祝动画

### 6.6 IP形象设计

**角色名称**：小勇（Xiao Yong）

**形象特征**：
- 拟人化小动物（建议：小老虎或小狮子，象征勇气和力量）
- 大眼睛、圆脸、短四肢，符合儿童审美
- 穿着运动装，头戴发带
- 表情丰富：开心、努力、鼓励、庆祝

**应用场景**：
- 启动页：小勇做热身运动
- 加载中：小勇跑步/跳绳动画
- 引导页：小勇示范动作
- 反馈：小勇表情变化（努力→开心→庆祝）
- 成就：小勇戴上皇冠/奖牌

## 八、数据隐私与合规方案

### 8.1 法规依据

- 《中华人民共和国个人信息保护法》
- 《儿童个人信息网络保护规定》
- GDPR（如考虑欧盟市场）
- COPPA（如考虑美国市场）

### 8.2 隐私设计原则

1. **数据最小化**：仅收集必要的年龄（用于分龄内容）、运动数据、视力数据
2. **本地优先**：运动视频、照片仅本地存储，不上传云端
3. **实时处理**：摄像头数据实时分析，不保留原始视频帧
4. **家长控制**：所有儿童数据收集需家长知情同意
5. **数据可删除**：家长可随时申请删除孩子全部数据

### 8.3 数据分类与处理

| 数据类型 | 收集方式 | 存储位置 | 加密 | 保留期限 |
|---------|---------|---------|------|---------|
| 运动记录 | 自动 | 本地+云端 | 是 | 用户注销后30天删除 |
| 骨骼关键点 | 实时计算 | 仅内存 | - | 不保留 |
| 视频/照片 | 用户授权 | 仅本地 | 是 | 用户控制 |
| 视力数据 | 家长录入 | 本地+云端 | 是 | 用户注销后30天删除 |
| 位置数据 | GPS | 本地 | 是 | 仅保留模糊化后的户外时间统计 |
| 设备信息 | 自动 | 云端 | 否 | 用户注销后30天删除 |

### 8.4 家长同意机制

1. **首次启动**：
   - 儿童模式首次启动需家长扫码绑定
   - 家长端完成实名验证（手机号+短信）
   - 展示隐私政策摘要（儿童友好版）
   - 家长签署电子同意书

2. **敏感操作**：
   - 摄像头首次使用需家长确认
   - 位置权限需家长确认
   - 数据上传云端需家长确认

3. **定期提醒**：
   - 每3个月提醒家长查看隐私设置
   - 重大功能更新时重新确认

### 8.5 合规审查清单

- [ ] 隐私政策完整版（法律版）
- [ ] 隐私政策摘要版（儿童友好版）
- [ ] 用户协议

- [ ] 数据收集清单（含目的、方式、期限）
- [ ] 第三方SDK隐私审查（每个SDK的隐私政策、数据收集范围）
- [ ] 数据安全影响评估报告
- [ ] 数据删除流程测试
- [ ] 数据导出功能（满足用户数据可携带权）
- [ ] 安全事件应急响应预案
